# Quy ước đa ngôn ngữ (i18n) — WTCC

## Tổng quan

WTCC hỗ trợ 2 ngôn ngữ: **English (en)** và **Tiếng Việt (vi)**. Hệ thống i18n hoạt động phía client, cấu hình ngôn ngữ lưu server-side (field `language` trong `config.json`).

## Kiến trúc

```
config.json (language: "en" | "vi")
       │
       ▼
public/js/i18n.js ─── Từ điển EN + VI
       │                    │
       ▼                    ▼
  Hàm t(key)         data-i18n* attributes
       │                    │
       ▼                    ▼
  Text động           Text tĩnh (HTML)
```

## File chính: `public/js/i18n.js`

File này chứa:

1. **Từ điển** — object chứa tất cả string cho mỗi ngôn ngữ:
   ```javascript
   const translations = {
     en: {
       'dashboard.title': 'Dashboard',
       'session.create': 'Create Session',
       // ...
     },
     vi: {
       'dashboard.title': 'Bảng điều khiển',
       'session.create': 'Tạo phiên',
       // ...
     }
   };
   ```

2. **Hàm `t(key)`** — tra cứu string theo key và ngôn ngữ hiện tại. Nếu key không tồn tại trong ngôn ngữ hiện tại, fallback sang `en`.

3. **Logic apply** — quét DOM tìm các element có `data-i18n*` attributes và apply text tương ứng.

## Cách sử dụng

### Text tĩnh (HTML)

Dùng `data-i18n` attribute trên element HTML:

```html
<!-- Nội dung text -->
<h1 data-i18n="dashboard.title"></h1>

<!-- Placeholder -->
<input data-i18n-placeholder="session.namePlaceholder">

<!-- Title/tooltip -->
<button data-i18n-title="session.killTooltip">✕</button>
```

Khi i18n apply, nội dung/attribute sẽ được thay bằng string từ từ điển.

### Text động (JavaScript)

Dùng hàm `t()` trong code:

```javascript
const message = t('session.created');
alert(t('error.nameInvalid'));
```

## Ngôn ngữ mặc định

- Mặc định: `"en"` (English)
- Lưu trong `config.json` field `language`
- Thay đổi qua Settings UI hoặc API `PUT /api/settings`

## Quy tắc khi thêm text mới

**BẮT BUỘC** thêm cả 2 ngôn ngữ EN + VI khi thêm bất kỳ text nào:

1. Mở `public/js/i18n.js`
2. Thêm key mới vào **cả** object `en` và `vi`
3. Dùng format key phân cấp: `section.action` hoặc `section.subsection.detail`
4. Trong HTML: dùng `data-i18n="key"` (hoặc `data-i18n-placeholder`, `data-i18n-title`)
5. Trong JS: dùng `t('key')`

### Ví dụ thêm text mới

```javascript
// Trong translations.en:
'logs.deleteConfirm': 'Delete this log file?',

// Trong translations.vi:
'logs.deleteConfirm': 'Xoá file log này?',
```

```html
<!-- Trong HTML -->
<span data-i18n="logs.deleteConfirm"></span>
```

## Quy ước đặt key

| Prefix | Dùng cho |
|---|---|
| `dashboard.*` | Trang dashboard |
| `terminal.*` | Trang terminal |
| `session.*` | Thao tác phiên |
| `settings.*` | Trang cài đặt |
| `auth.*` | Đăng nhập/đăng xuất |
| `error.*` | Thông báo lỗi |
| `confirm.*` | Hộp thoại xác nhận |
| `logs.*` | Quản lý log |
| `warning.*` | Cảnh báo cấu hình |

## Lưu ý

- **Không hardcode text** trực tiếp trong HTML hay JS. Luôn dùng i18n key.
- **Fallback**: Nếu key chưa có bản dịch VI, hệ thống hiển thị bản EN (không crash).
- **Không dùng template literal phức tạp** trong từ điển. Nếu cần interpolation, dùng placeholder đơn giản và replace trong code.
- **Ngôn ngữ áp dụng toàn cục** — tất cả client cùng thấy 1 ngôn ngữ (theo config server). Không hỗ trợ per-user language.
